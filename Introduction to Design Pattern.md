# Introduction
An architectural design pattern is a proven, industry-accepted solution to the design problem. The architectural design pattern allows you to develop an efficient enterprise-grade software solutions. In the context of a software design and development, a design pattern can be classified into the following types:

## Creational Design Pattern
The creation design pattern creates an object in such a way that it can be decoupled from its implementation system. Or, in other words, the creational design pattern is a process to create objects. Thus, these objects are efficient, flexible, easily organized, and can be decoupled from their implementation system. 

A few examples of the creational design patterns are Singleton (one instance), Factory Method (subclass instantiation), Abstract Factory (families of objects), Builder (step-by-step construction), and Prototype (cloning objects).

## Structural Design Pattern
The structural design pattern is a design template. It allows you to assemble and manage several objects and classes so that you can create a robust and more efficient software system. You can use these disparate objects and classes to create a software system.

A few examples of structural design pattern are Adapter (connecting incompatible interfaces), Facade (simplifying complex systems), Composite (tree structures), and Decorator (adding behavior dynamically).

## Behavioral Design Pattern
The behavioral design pattern is a proven and tested design template that allows you to establish the communication and interaction between different objects and classes in the software system. It also manages the assignment of responsibilities among different components. This design pattern is best-suited to define complex control flow in the software application.

The examples of the behavioral design pattern are Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Template Method, and others.
