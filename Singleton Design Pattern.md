# Singleton Design Pattern
The Singleton design pattern is a creational pattern that ensures a class has only one instance while providing a global access point to it. The programmer defines a constructor by using new keyword. The scope of the constructor is kept private or restricted. He creates a single instance with the constructor. A remote system can access this instance. This design pattern is useful to design centralized services.

It tends to make system tightly coupled. Its overuse can reduce modularity and scalability. In the context of technical term, the Singleton architecture is only a design pattern. It is not a full architectural design pattern. However, it influences software architecture because it controls object lifecycle and global access behavior.
